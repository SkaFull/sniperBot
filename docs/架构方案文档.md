## 项目概述

### 1.1 项目背景

Solana 狙击机器人是一个高性能的自动化交易系统,专门用于在 Solana 链上捕捉新代币发行机会。在 Solana 这种高性能区块链上,手动交易几乎无法盈利,因为:

- **速度鸿沟**: 人类操作需要 5-15 秒,而机器人只需 10-200 毫秒
- **竞争激烈**: 全球顶尖量化团队、黑客和诈骗分子都在竞争
- **高风险**: 99% 的新币最终归零,需要严格的风控系统

### 1.2 项目目标

- **极速响应**: 在流动性添加的同一区块内完成买入(毫秒级)
- **安全防护**: 自动识别并规避貔貅盘、Rug Pull 等骗局
- **智能卖出**: 自动化止盈止损策略,最大化收益
- **稳定运行**: 24/7 全天候监控,高可用性架构

### 1.3 核心功能

1. **实时监听**: 监听 Pump.fun、Raydium、Meteora 等平台的流动性添加事件
2. **安全审计**: 自动检测代币的 Mint Authority、Freeze Authority、LP 锁定状态等
3. **交易构建**: 构建包含优先费和 Jito Bundle 的原子交易
4. **智能卖出**: 支持硬止盈止损、移动止损、时间止损等多种策略
5. **仓位管理**: 自动化仓位分配和资金归集

#### 2.2.3 Security 模块 (安全模块)

**职责**: 代币安全审计,过滤风险代币

**核心功能**:

- Mint Authority 检查
- Freeze Authority 检查
- LP 锁定状态检查
- 筹码分布分析
- 交易模拟验证

**安全评分卡**:

| 检查项              | 安全标准    | 风险等级        |
| ---------------- | ------- | ----------- |
| Mint Authority   | null    | 🔴 高风险(未放弃) |
| Freeze Authority | null    | 🔴 高风险(貔貅盘) |
| LP 锁定            | >95% 销毁 | 🟠 中风险(可撤池) |
| 前10持仓            | <30%    | 🟠 中风险(老鼠仓) |
| 交易模拟             | 成功      | 🔴 高风险(貔貅)  |

**代码实现**:

```javascript
async function checkTokenSecurity(connection, mintAddress) {
  const mintInfo = await getMint(connection, mintAddress);
  
  // 检查 Mint Authority
  if (mintInfo.mintAuthority !== null) {
    return { safe: false, reason: "Mint 权限未放弃" };
  }
  
  // 检查 Freeze Authority
  if (mintInfo.freezeAuthority !== null) {
    return { safe: false, reason: "存在冻结权限(貔貅盘)" };
  }
  
  // 检查 LP 锁定状态
  const lpLocked = await checkLPLocked(mintAddress);
  if (lpLocked < 95) {
    return { safe: false, reason: "LP 未锁定" };
  }
  
  // 检查筹码分布
  const top10Holders = await getTop10Holders(mintAddress);
  if (top10Holders > 30) {
    return { safe: false, reason: "筹码集中度高" };
  }
  
  // 交易模拟
  const simulation = await simulateTransaction(mintAddress);
  if (simulation.err) {
    return { safe: false, reason: "交易模拟失败" };
  }
  
  return { safe: true };
}
```

#### 2.2.4 Executor 模块 (执行模块)

**职责**: 构建并发送交易,完成买入操作

**核心功能**:

- 计算预算设置
- Swap 指令构建
- Jito Bundle 打包
- 交易签名与发送
- 失败重试机制

**交易构建流程**:

```
1. 获取最新 Blockhash
2. 设置 Compute Unit Limit
3. 设置 Compute Unit Price
4. 构建 Swap 指令
5. 构建 Jito Tip 指令
6. 打包成 Versioned Transaction (v0)
7. 签名交易
8. 发送到 Jito Block Engine
```

**Jito Bundle 优势**:

- **原子性**: 买入和小费要么全部成功,要么全部失败
- **防夹子**: 通过私密通道发送,MEV 机器人无法看到
- **失败不付费**: 买入失败时,小费也不会被扣除

**代码实现**:

```javascript
async function buildAndSendBundle(tokenMint) {
  // 1. 获取 Blockhash
  const { blockhash } = await connection.getLatestBlockhash();
  
  // 2. 构建指令
  const instructions = [
    ComputeBudgetProgram.setComputeUnitLimit({ units: 80000 }),
    ComputeBudgetProgram.setComputeUnitPrice({ microLamports: 150000 }),
    await createSwapInstruction(tokenMint),
    createJitoTipInstruction()
  ];
  
  // 3. 打包交易
  const messageV0 = new TransactionMessage({
    payerKey: wallet.publicKey,
    recentBlockhash: blockhash,
    instructions
  }).compileToV0Message();
  
  const transaction = new VersionedTransaction(messageV0);
  transaction.sign([wallet]);
  
  // 4. 发送 Bundle
  const bundleId = await jitoClient.sendBundle([transaction]);
  
  return bundleId;
}
```

### 3.1 交易生命周期管理

#### 3.1.1 Solana 节点角色

```
┌──────────────┐
│  RPC 节点    │ ← 接收请求,转发交易
│  (前台接待)  │
└──────┬───────┘
       │
       │ Gossip 协议
       │
┌──────▼───────┐
│ Validator    │ ← 验证交易,投票确认
│  (记账员)    │
└──────┬───────┘
       │
       │ Leader Schedule
       │
┌──────▼───────┐
│   Leader     │ ← 打包区块(400ms时间槽)
│  (当值班长)  │
└──────────────┘
```

#### 3.1.2 交易传播路径

**传统路径(慢)**:

```
机器人 → RPC(HTTP) → Gossip广播 → Leader
延迟: 200-500ms
```

**狙击路径(快)**:

```
机器人 → Leader TPU(UDP) → 直接打包
延迟: 10-50ms
```

**Jito 路径(最快+安全)**:

```
机器人 → Jito Block Engine → Jito Validator
延迟: 10-30ms + 防夹子
```

### 3.2 优先费策略

#### 3.2.1 费用计算公式

```
Total Fee = Base Fee(5000) + Priority Fee
Priority Fee = Compute Unit Limit × Compute Unit Price
```

#### 3.2.2 动态费用策略

```javascript
async function calculatePriorityFee() {
  // 1. 模拟交易获取精确 CU 消耗
  const simulation = await connection.simulateTransaction(tx);
  const cuConsumed = simulation.value.unitsConsumed;
  
  // 2. 设置 CU Limit(留余量)
  const cuLimit = cuConsumed * 1.2;
  
  // 3. 查询网络中位数
  const fees = await connection.getRecentPrioritizationFees();
  const medianFee = calculateMedian(fees);
  
  // 4. 加价策略
  const price = medianFee * 2;
  
  return { cuLimit, price };
}
```

### 3.3 WebSocket 监听优化

#### 3.3.1 承诺级别选择

| 承诺级别      | 延迟        | 可靠性     | 适用场景     |
| --------- | --------- | ------- | -------- |
| processed | \~0ms     | 低(可能分叉) | **狙击必须** |
| confirmed | 400-800ms | 中       | 普通转账     |
| finalized | 1-2s      | 高       | 重要交易     |

#### 3.3.2 双跳问题解决方案

**问题**: WebSocket 日志不包含 Token Mint 地址

**慢速方案**:

```
WebSocket 推送 → HTTP getTransaction → 解析 Mint
延迟: 200ms+
```

**极速方案(Geyser gRPC)**:

```
gRPC Stream → 直接包含 AccountKeys → 本地解析
延迟: 0ms
```

**Geyser 代码示例**:

```javascript
import Client from "@triton-one/yellowstone-grpc";

const client = new Client("grpc-endpoint", "auth-token");
const stream = await client.subscribe();

stream.write({
  transactions: {
    pumpFunFilter: {
      accountInclude: ["6EF8rrecthR5Dkzon8Nwu78hRvfCKubJ14M5uBEwF6P"]
    }
  }
});

stream.on("data", (data) => {
  if (data.transaction) {
    // 直接获取所有账户地址
    const accountKeys = data.transaction.transaction.message.accountKeys;
    const mintAddress = accountKeys[1]; // 无需第二次请求

#### 6.1.1 硬编码风险检查

async function checkHardcodedRisks(mintAddress) {
  const mintInfo = await getMint(connection, mintAddress);
  
  const risks = [];
  
  // 1. Mint Authority
  if (mintInfo.mintAuthority !== null) {
    risks.push({
      level: "HIGH",
      type: "MINT_AUTHORITY",
      message: "项目方可无限增发代币"

  // 2. Freeze Authority
  if (mintInfo.freezeAuthority !== null) {
    risks.push({
      level: "HIGH",
      type: "FREEZE_AUTHORITY",
      message: "项目方可冻结账户(貔貅盘)"
    });
  }
  
  return risks;
}
```

#### 6.1.2 流动性陷阱检查

```javascript
async function checkLiquidityTrap(mintAddress) {
  // 检查 LP Token 是否锁定
  const lpTokenAccount = await getLPTokenAccount(mintAddress);
  
  // LP Token 锁定方式:
  // 1. 销毁(Burn): 发送到 dead 地址
  // 2. 锁定(Lock): 发送到锁定合约
  
  const burntAmount = await checkBurntLP(lpTokenAccount);
  const lockedAmount = await checkLockedLP(lpTokenAccount);
  
  const totalLocked = burntAmount + lockedAmount;
  const lockedPercentage = totalLocked / totalSupply * 100;
  
  if (lockedPercentage < 95) {
    return {
      safe: false,
      reason: "LP 锁定比例过低,可能撤池跑路"
    };
  }
  
  return { safe: true };
}
```

async function executeWithRetry(tx, maxRetries = 3) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      const sig = await connection.sendTransaction(tx);
      const confirmed = await connection.confirmTransaction(sig, "processed");
      
      if (confirmed.value.err) {
        throw new Error("Transaction failed");
      }
      
      return sig;
    } catch (error) {
      log.warn(`第 ${i+1} 次尝试失败: ${error.message}`);
      
      // 更新 Blockhash 和优先费
      tx.recentBlockhash = await getCachedBlockhash();
      tx.add(ComputeBudgetProgram.setComputeUnitPrice({
        microLamports: await getCachedPriorityFee()
      }));
      
      // 等待 100ms 后重试
      await sleep(100);
    }
  }
  
  throw new Error("所有重试失败");
}

### D. 错误处理指南

| 错误类型                 | 原因     | 解决方案           |
| -------------------- | ------ | -------------- |
| `Blockhash expired`  | 区块哈希过期 | 更新 Blockhash   |
| `Insufficient funds` | 余额不足   | 归集资金或减少买入金额    |
| `Slippage exceeded`  | 滑点过低   | 提高滑点设置         |

| `Transaction failed` | 交易执行失败 | 检查安全审计或提高优先费   |
